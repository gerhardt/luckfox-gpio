Help Page & Command Reference

SCPI Command Set:

 

Connect via telnet to {{ ipa }}:5025 for computer (e.g. Python) control

 

*IDN?: Ask the device for its identity
*RST: Reset the device
SYST:ERR?: check the error status
GPIO:LIST?: list which GPIOs are listed in the source code
GPIO:SETUP: setup the different channels
GPIO:WRITE: write a channel
GPIO:READ?: read a channel
GPIO:TOGGLE: toggle a channel
GPIO:STATUS?: check the status of the different channels

Example Programm:

 

 

#!/usr/bin/env python3

import socket
import time

TCP_IP = '{{ipa}}'
TCP_PORT = 5025
BUFFER_SIZE = 1024

s = socket.socket(socket.AF_INET, socket.SOCK_STREAM)
s.connect((TCP_IP, TCP_PORT))
s.send(b"*IDN?\r\n")
data = s.recv(BUFFER_SIZE)
print("received data:", data)
s.send(b"GPIO:WRITE 54,1\r\n")
time.sleep(1)
s.send(b"GPIO:WRITE 55,1\r\n")
time.sleep(1)
s.send(b"GPIO:WRITE 54,0\r\n")
time.sleep(1)
s.send(b"GPIO:WRITE 55,0\r\n")
time.sleep(1)
s.send(b"GPIO:TOGGLE 54\r\n")
time.sleep(1)
s.send(b"GPIO:TOGGLE 55\r\n")
time.sleep(1)
s.close()
LuckFox Pinout:

 

Pinout calculation:

 

Add more pins as needed - refer to LuckFox pinout diagram

Pin calculation: bank * 32 + (group * 8 + X)

Example: GPIO1_C7 = 1 * 32 + (2 * 8 + 7) = 55

 

This all needs to be edited in the original file: gpio_server.py on the luckfox-device. You can login there with ssh pico@{{ ipa }}. The gpio_server.py file is in the users home-directory.

 

When you want to run the file on the platform you have to first kill the running webinterface. You can find the process with ps ax | grep gpio. Then you kill the process by using sudo kill [process id]. After this you can launch the command sudo /usr/bin/python3 /home/pico/gpio_server.py.
